Version 4.2.0  (2026-06-25)
--------------------------------------------------------------------------------
Feature release — Keypad module only. Adds triple/treble press and an optional
relative-rotary (scroll-wheel / dial) endpoint. Only the Keypad symbol moves to
the v4.2 generation; the other 13 modules are unchanged and stay v4.1 (their
symbols remain compatible with the 4.2.0.0 library).

Keypad — triple/treble press (buttonevent 005):
  - New DIGITAL_OUTPUT Button_triple_press_fb[1..N] (500 ms pulse) for deCONZ
    buttonevent code 005 (e.g. the IKEA BILRESA / E2490 wheel click). The
    ButtonEvent map gains TreblePress = 5.

Keypad — optional ZHARelativeRotary endpoint (scroll wheel / dial):
  - New optional STRING_PARAMETER Rotary_UniqueID. When set, the module also
    registers that uniqueid with the broker and parses its rotation events
    (WS only, transient — never re-fired from an HTTP GET).
  - New outputs: Rotate_cw / Rotate_ccw (500 ms pulses), Rotation_fb
    (expectedrotation, SIGNED: + = CW, - = CCW; read as signed in SIMPL+),
    Rotary_event_fb (rotaryevent) and Rotation_duration_fb
    (expectedeventduration, ms).
  - One physical scroll-wheel remote (e.g. IKEA BILRESA) is a single symbol:
    the wheel click is a buttonevent on the ZHASwitch endpoint, the rotation is
    the separate ZHARelativeRotary endpoint; battery/online/identity are shared.

Keypad — rotary level for a volume / dimmer (time-based ramp):
  - New ANALOG_OUTPUT Level_fb (0-65535 = 0-100%). The ZHARelativeRotary dial
    reports a fixed rate (~1 event/s while moving; rotaryevent 1 = moving,
    2 = stop) with no speed magnitude, so the level is RAMPED smoothly over time
    while the wheel is turned, in the direction of the rotation sign, and stops
    on rotaryevent 2 (or a short no-event timeout).
  - New INTEGER_PARAMETER Full_Scale_Seconds: seconds of continuous turning for
    0-100% (default 4).
  - New ANALOG_INPUT Set_Level (0-65535 = 0-100%) presets the level so the dial
    tracks the real device value (e.g. wire the device's current volume in).

BREAKING (Keypad symbol only):
  - Rotary_UniqueID and Full_Scale_Seconds add parameters, so the keypad
    symbol's leading _SKIP_ count and the new analog/digital signals shift its
    layout. #SYMBOL_NAME and the .usp
    filename now carry "v4.2" (DeConz_Keypad v4.2 /
    DeConz_Keypad_Wrapper_v4.2.usp). Replace the v4.1 keypad symbol with the v4.2
    one and re-check its connections. The other modules are not affected.

Build:
  - AssemblyVersion / AssemblyFileVersion 4.2.0.0.

Version 4.1.1  (2026-06-25)
--------------------------------------------------------------------------------
Bugfix release (library only — no wrapper/signal changes).

FireStr truncation crash fixed (Power, Valve, Shade, Alarm, Motion, GroupControl,
Scene):
  - FireStr truncated over-long strings with s.Substring(0, 65000) instead of
    s.Substring(0, 250). For any string between 251 and 64999 characters this
    threw ArgumentOutOfRangeException. In practice it fired whenever Enable_Debug
    was on (DebugLog passes a 251-character "…"-trimmed message to FireStr): the
    uncaught exception aborted parsing of the WebSocket event, so the affected
    module stopped updating its outputs while debug was enabled. The other five
    modules already used the correct 250 cap. Build a new DeConzZigbee.clz; no
    wrapper recompile or demo re-wiring is required.

Build:
  - AssemblyVersion / AssemblyFileVersion 4.1.1.0.

Version 4.1.0  (2026-06-25)
--------------------------------------------------------------------------------
Feature release. Adds the missing LastAnnounced output to the last three modules,
makes Get_State re-send static device info, and introduces a global/local
"permanent string re-assert" feature (the internal equivalent of the SIMPL
"Make String Permanent" symbol). New wrapper signals shift the symbol layout, so
this carries the v4.1 symbol generation.

BREAKING - SIMPL Windows symbol rename:
  All 14 module symbols move to the v4.1 generation. #SYMBOL_NAME and the .usp
  filenames now carry "v4.1" (e.g. "DeConz_Gateway v4.1",
  DeConz_Gateway_Wrapper_v4.1.usp). Existing v4.0 symbols must be replaced with
  the v4.1 symbols; re-check signal connections after the swap (see the signal
  shifts below).

LastAnnounced output added (Power, Alarm, Motion):
  - New STRING_OUTPUT LastAnnounced_fb, placed directly after LastSeen_fb (as on
    all other modules). Parses the top-level "lastannounced" timestamp. This
    shifts the following string outputs (Manufacturer/ModelId/Name/raw JSON) down
    by one position on these three modules.

Get_State now refreshes static device info (all device-info modules):
  - Contact, Climate, Valve, Motion, Power, Keypad, Shade, Thermostat, LightWs,
    Alarm: a Get_State pulse now re-sends Manufacturer/ModelId/Name (and the
    other once-only identity strings), which were previously emitted only once
    per online session. Internally the _staticInfoSent guard is reset at the
    start of GetState().

Permanent string re-assert ("Make String Permanent" equivalent):
  - While enabled, each device-info module periodically re-fires its cached
    (non-raw, non-debug) string outputs, so late-joining sinks (panel reconnect,
    program restart) always see the current values.
  - Gateway: new DIGITAL_INPUT Permanent_Resend_Enable (global, acts on ALL
    modules) and new INTEGER_PARAMETER Permanent_Resend_Seconds (interval,
    default 30, range 1-3600). The new parameter shifts the gateway symbol's
    parameter and digital-input layout - re-check gateway connections.
  - Each device-info module: new local DIGITAL_INPUT Permanent_Resend_Enable,
    appended at the end of the digital-input block.
  - Effective state per module = Gateway enable OR local enable. The Gateway
    enable, when high, forces re-assert on all modules (overrides the locals);
    when low, each module's local enable applies. Routed AppDomain-wide via the
    static broker; the interval is read on every tick.

Central API key (BREAKING – parameter removed from every device module):
  - The deCONZ API key is now entered ONCE on the Gateway, as a new
    STRING_PARAMETER API_Key, and distributed to all device modules via the
    static broker (DeConzBroker.ApiKey). Device modules read it lazily when
    building their HTTP request URLs.
  - The per-module API_Key parameter has been removed from all 12 modules that
    use it (Power, Alarm, Climate, Keypad, GroupControl, Motion, Contact, Valve,
    Scene, LightWs, Shade, Thermostat). Removing this parameter shifts the
    symbol's leading _SKIP_ count down by one (the first DIGITAL_INPUT and
    DIGITAL_OUTPUT block) – re-check signal connections. GroupControl and Scene
    now have no parameters at all.
  - The Gateway gains the API_Key parameter; together with Permanent_Resend_
    Seconds this shifts the gateway symbol's parameter and signal layout.

Build:
  - AssemblyVersion / AssemblyFileVersion 4.1.0.0.

    Previously hardcoded.
  - Use_TLS (INTEGER_PARAMETER, 0 = plain ws://, 1 = TLS wss://): selects the

did not compile without these):
  - DeConzDevice: added the missing private _debugEnabled field (three
    debug-gated log statements referenced it).
  - DeConzKeypad: added the Fire(KeypadBoolDelegate, ushort) and

  - DeConzLightWs / DeConzShade / DeConzThermostat: fixed the "DebugDebugLog"

  - DeConzShade: fixed undefined identifier "raw" in the open/closed threshold

Version 3.6.0  (2026-06-18)
--------------------------------------------------------------------------------
New feature: stale-value reset (10 device modules).

  When no update is received from a device for more than one hour, the module
  now resets its live value and status outputs so the touch panel no longer
  shows outdated readings. A per-module timer checks every 5 minutes; the reset
  fires once and re-arms automatically as soon as fresh data arrives.

  RESET on stale (Bool/Analog -> 0, String -> ""):
    Alarm:      online, battery_low, Battery_state, Voltage, tampered, alarm,
                fire, carbonmonoxide
    Climate:    online, battery_low, Battery_state, Voltage,
                Temperature(+str), Humidity(+str), Pressure(+str)
    Contact:    online, open, closed, battery_low, Battery_state, Voltage
    Keypad:     online, battery_low, Battery_state
    Light:      online, On, Off, Brightness, Colortemp, X, Y, R, G, B,
                Hue, Sat, Colormode, Effect
    Motion:     online, battery_low, Battery_state, Voltage, presence,
                tampered, Lux, Lightlevel, dark, daylight
    Power:      online, battery_low, Battery_state, On, Off, Power, Voltage,
                Current, Consumption_Power
    Shade:      online, lowbattery, Battery_state, Voltage, Lift, Tilt,
                open, closed, moving
    Thermostat: online, lowbattery, Battery_state, Voltage, on, off, heating,
                Valve, Temperature(+str), windowopen_detected, errorcode
    Valve:      online, valve_on, valve_off, water_detected, battery_low,
                Battery_state, flow_volume

  PRESERVED on stale (never reset):
    Device identity (manufacturer, modelid, name, type, swversion, device id)
    Timestamps (lastseen, lastannounced)
    Capabilities (Sensitivity_Max, hascolor)
    Configuration / setpoints (thermostat setpoints, mode, offset, locked,
      schedule, windowopen_set, displayflipped, setvalve; motion sensor_on,
      led_indication, user_test, sensitivity, delay)
    Cumulative meter (Power consumption kWh)
    Raw JSON outputs and Debug$

  Threshold is reception-based (time since last received update), so no device
  clock or timezone handling is involved. GroupControl, Scene and Device are
  not affected (no per-device lastseen).

  No wrapper signal changes — drop-in at the SIMPL Windows level.

Version 3.5.0  (2026-06-10)
--------------------------------------------------------------------------------
Performance and memory hardening (code review follow-up). No wrapper signal
changes — drop-in replacement for v3.3.x at the SIMPL Windows level.

HTTP response disposal (all 12 device modules):
  All 31 HttpClientResponse callbacks now dispose the response in a finally
  block. Previously the response object (holding a stream/handle) was never
  released, leaking handles over long-term operation with periodic polling.

Debug logging cost eliminated when debug is off (all modules, 43 call sites):
  Payload-carrying DebugLog calls (full WS JSON / HTTP body) are now guarded
  with `if (_debugEnabled)` at the call site. Previously the entire JSON
  payload was concatenated into a new string on every event even when debug
  output was disabled, then immediately discarded — significant avoidable GC
  pressure on busy sensors. The [RX] log in the gateway (fired on every frame)
  is included.

Static device-info parsed once instead of every event (10 modules):
  Manufacturer, model id, name, type, swversion (and hascolor/groups for
  Light) were re-extracted on every WebSocket state event via full-string
  scans, despite never changing in practice. These are now published once
  (guarded by a _staticInfoSent flag) and re-published a single time after a
  reconnect (reset in ScheduleGetState / ScheduleGetAll, or on online-timeout
  for sensor-only modules). lastseen / lastannounced timestamps continue to
  fire on every event since they genuinely change. This removes 5-6 full JSON
  scans per event on every device.

No functional signal behaviour changes.

Version 3.3.1  (2026-05-30)
--------------------------------------------------------------------------------
Fixed _SKIP_ placement in all 14 SIMPL+ wrappers per corrected rule:
  Leading _SKIP_ entries are required only in the FIRST input block
  (DIGITAL_INPUT) and FIRST output block (DIGITAL_OUTPUT) to push signal
  names below the parameter fields in SIMPL Windows. All inputs share one
  left column; all outputs share one right column. ANALOG_INPUT, STRING_INPUT,
  ANALOG_OUTPUT and STRING_OUTPUT continue in the same column below the already-
  cleared region and require no leading skips — these were spurious dead rows.

  Removed leading _SKIP_ from AI, AO, SI, SO blocks in all 14 modules.

Additional cleanup:
  Gateway   DI: double _SKIP_ after Disable_Websocket reduced to single.
  Keypad    DO: trailing _SKIP_ at end of block removed.
  Light_WS  DI: SET_RGB moved into On/Off group (Set_On, Set_Off, SET_RGB)
               rather than standing alone between two skips.
  Light_WS  SO: redundant and double _SKIP_ entries removed; simplified to
               one separator between color/device info and raw JSON outputs.

No C# changes. No functional changes.

Version 3.0.0  (2026-05-26)   *** FIRST PUBLIC RELEASE ***
--------------------------------------------------------------------------------
First publicly released version of the DeConz Zigbee SIMPL# module suite.

This release consolidates the complete module set and all fixes accumulated
during private development into a stable, documented public package.

Included modules (11):
  Gateway, Device, Keypad, Light_WS, Valve, Shade, Thermostat,
  Climate, Contact, Group_Control, Scene.
Supporting infrastructure:
  DeConzBroker (static multicast bus), DeConzJsonParser (shared parser),
  DeConzWebSocketClient (RFC-6455 gateway connection).

Changes in this release:
  - Signed temperature output fix (Climate, Thermostat):
    The Temperature_fb analog output now carries negative temperatures as a
    16-bit two's-complement value, to be read as a signed analog in SIMPL
    (e.g. -5.0 C → 65036). Previously negative temperatures were clamped to 0.
    The Temperature_str string output was and remains correct for all values.
    Humidity and pressure are unsigned and unchanged.
  - Documentation rewritten in English, restructured, and brought up to date
    for all 11 modules.

Version 2.19.0  (2026-05-24)
--------------------------------------------------------------------------------
Memory management: fixed fire-and-forget CTimer leak (critical code review).
  - 12 one-shot CTimer instances across 8 modules were created without being
    stored or disposed (ScheduleGetState/ScheduleGet stagger timers and
    Scene success pulses).
  - Crestron CTimer is IDisposable and holds an internal .NET timer +
    callback delegate; one-shot timers do not free themselves after firing.
  - Fix: each one-shot timer is now held in a local variable and disposes
    itself at the end of its callback:
      CTimer t = null;
      t = new CTimer(_ => { Work(); if (t != null) t.Dispose(); }, null, delay);
  - Affected: Climate, Contact (×2), Keypad, LightWs, Scene (×2),
    Shade (×2), Thermostat (×2), Valve.
  - Stored timers (_onlineTimer, _pollTimer, _brightnessTimer, _moveTimer,
    _aliveTimer, _reconnectTimer) were already correctly stopped/nulled and
    are unchanged.

Code review notes (no change required):
  - Dispose() methods exist and are symmetric (Register/Unregister balanced)
    but are not called by SIMPL+ wrappers. Acceptable: SIMPL# objects live
    for the whole program lifetime; AppDomain unload frees everything on
    program restart. Methods retained for completeness.
  - HttpClient is a long-lived readonly instance per module – this is the
    recommended usage pattern, not a leak.
  - WebSocketClient socket cleanup correctly removes event handlers (-=) and
    nulls references in CloseSocket().

Version 2.0.0  (2026-05-20)  [initial SIMPL# rewrite]
--------------------------------------------------------------------------------
Full rewrite of original SIMPL+ deCONZ gateway as SIMPL# module suite.

DeConzBroker (static class):
  - Central message bus; routes WS payloads to registered device modules.
  - No signal wiring between Gateway and device modules required.
  - GatewayIP property for HTTP URL construction by device modules.
  - Thread-safe dispatch via CrestronInvoke.BeginInvoke.

DeConzWebSocketClient:
  - Native RFC-6455 WebSocket (TCP via CrestronTCP / SecureTCPClient for TLS).
  - Full frame decoder: fin/opcode/mask/payload length (including 16- and
    64-bit extended lengths).
  - Exponential back-off reconnect (5 s → 30 s max).
  - Ping/Pong keepalive.
  - TLS/WSS support via Use_TLS parameter.
  - Traffic-present indicator with configurable alive timeout.

DeConzDevice:
  - Raw JSON passthrough; forwards WS payloads to SIMPL+ raw_json_out.
  - Registers with broker by uniqueid (no slot index required initially).

Improvements over original SIMPL+ module:
  - Reconnect logic: none → exponential back-off
  - TLS: not supported → SecureTCPClient with Use_TLS parameter
  - Frame decoder: fragile string scan → full RFC-6455 parser
  - Device routing: STRING_OUTPUT Device[] array → static broker
  - IP wiring: Buffer_Input to every module → DeConzBroker.GatewayIP
  - HTTP library: external HTTP_Command_Library.clz → native HttpClient
  - Keypad 2D array bug: Button_long_release_fb[N,N] → corrected to [N]
  - Online timeout: hardcoded → configurable parameter
  - Threading: SIMPL+ event loop → CrestronInvoke worker threads
